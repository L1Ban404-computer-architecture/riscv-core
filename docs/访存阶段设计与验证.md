# 访存阶段设计与验证

MEM 把 EX/MEM 访存事务转换为顺序 CoreBus 请求，并负责 store lane 对齐、load
提取、访问错误转换和 MEM/WB 边界。

## 单 outstanding 结构

数据侧固定一个在飞事务：

```text
EX/MEM -> CoreBus request
     \-> fall-through outstanding slot

CoreBus response + slot -> response merge -> MEM/WB register
```

请求握手与 outstanding slot 接受是同一个事件。fall-through 行为允许请求和
响应零延迟同周期完成。load/store 均等待响应；slot 有效时年轻访存和非访存不能
越过当前事务。

pending-load 接口只提供 `valid` 和目标寄存器地址。响应进入 MEM/WB 后，数据从
正常 MEM/WB 前递路径提供；尚不存在的数据不再伪装成写回事务传递。

## 数据与错误

- 地址保留完整有效地址和 size；
- store 根据地址低位生成 lane 对齐 wdata/wstrb；
- load 根据保存的地址低位、size 和 sign-ext 提取返回字；
- error 转换为 load/store access fault，`tval` 为原始有效地址；
- faulting load 清除写回，faulting 访存不报告成功内存事件；
- error 同周期阻止年轻请求交接，直到异常在 WB 提交。

多 outstanding 不能通过参数恢复：若未来需要更高并发，必须设计 WB 提交式
store buffer 或具有撤销能力的总线事务。

## 验证

测试覆盖普通事务旁路、MEM/WB 背压、全部 byte/halfword lane、load 符号扩展、
零延迟响应、单 outstanding 反压、pending load 和访问错误交接边界。统一运行方式
见 `README.md`。
