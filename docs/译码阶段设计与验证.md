# 译码阶段设计与验证

ID 消费完整的 IF/ID 事务，完成指令译码、立即数生成和寄存器读取，并通过单入口
`stream_register` 输出 ID/EX 事务。阶段边界统一使用 ready/valid；flush 同周期
屏蔽输入握手并清除寄存器中的年轻事务。

## 数据与控制

```text
IF/ID instruction -> decoder -> execute_ctrl_bus_t
                  -> imm_gen -> immediate
                  -> regfile -> rs1/rs2 values
                         |
                         v
                    ID/EX register
```

`decoder` 输出寄存器地址、立即数类型、执行控制和独立的非法标志。合法控制直接使用
`execute_ctrl_bus_t`，非法状态不再作为控制字段重复进入流水线。ID 将非法指令、
ECALL 和 EBREAK 转换为 `exception_bus_t`；已有的取指异常年龄更老，不能被覆盖。

支持 RV32I 整数、跳转、分支、load/store、FENCE，以及六条 Zicsr 指令和 MRET。
FENCE.I 未实现。FENCE 的 `rs1/rd` 与保留的 `fm/pred/succ` 配置均按 RV32I 要求
忽略，并在当前严格顺序数据通路上作为保守全栅栏退休。

decoder 只检查 Zicsr 的编码语法。CSR 地址实现性由 EX 使用 `csr_unit` 组合读端口
判断，使受支持 CSR 列表只有一个功能来源。

## 寄存器与流控

寄存器堆不复位普通 GPR；x0 的读值固定为零且写入被忽略。WB 与 ID 同周期访问同一
寄存器时显式旁路写回值，避免依赖阵列 read-during-write 语义。

`serialize_block_i` 阻止年轻指令越过正在执行的 CSR、SYSTEM 或异常事务。ID/EX
寄存器支持满载同拍替换，不在连续普通指令之间插入气泡。

## 验证

`tests/id_stage/test_id_stage.py` 覆盖代表性数据通路、全部主要 opcode 的合法性矩阵、
立即数、WB 同拍旁路、FENCE 保留字段、未知 CSR 的语法译码，以及背压和同拍替换。
整核异常与 CSR 行为由 `tests/riscv_core/test_riscv_core.py` 检查。

运行方式统一由仓库顶层提供：

```sh
make test ALL=id_stage
```
